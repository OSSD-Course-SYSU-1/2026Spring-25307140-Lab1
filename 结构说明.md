鸿蒙DecodePlayControl项目 完整文件&目录详细说明

一、核心业务源码

1. entry
应用主入口模块，整个APP的启动起点，存放所有ArkTS业务代码、页面布局、交互逻辑、功能实现，是项目最核心的程序源码主体。
2. AppScope
项目全局根目录，存放整个应用的全局公共资源、全局样式、多语言文本、应用基础声明，所有模块都可以共享调用这里的内容。
3. src
自定义公共源码目录，用来存放通用工具函数、封装工具类、公共业务方法、通用工具代码。
4. screenshots/device
项目演示截图、展示素材文件夹，存放项目运行效果配图。

二、项目基础核心配置文件

1. oh-package.json5
鸿蒙OHPM包管理配置，项目的依赖清单，声明所有需要安装的第三方开源库、组件版本，别人拉取项目靠这个文件还原全部依赖。
2. oh-package-lock.json5
依赖锁定文件，精准锁定每一个依赖的具体版本，保证所有人、所有电脑编译环境完全一致。
3. build-profile.json5
工程全局编译配置，定义项目模块结构、编译基础参数、工程全局规则。
4. hvigorfile.ts
项目全局Hvigor构建脚本，自定义项目编译、打包、构建流程规则。
5. hvigor-config.json5
Hvigor构建工具全局配置参数文件。
6. obfuscation-rules.txt
代码混淆规则文件，正式发布打包时压缩、混淆源码，防止代码被反编译、保护知识产权。
7. .gitignore
Git忽略规则配置文件，定义哪些缓存、垃圾、本地文件不需要提交、上传到远程仓库。
8. LICENSE
项目开源许可协议声明文件，定义代码开源使用权限。
9. OAT.xml
鸿蒙工程属性、权限、基础工程配置描述文件。
10. README.md
项目主页介绍文档，介绍项目功能、使用方法、部署运行教程，GitHub仓库首页默认展示。
11. 结构说明.md
本文档，项目全部目录、文件的详细作用注释说明。
12. modules.xml / vcs.xml / workspace.xml
工程模块索引、版本控制基础配置文件。

三、.hvigor 编译缓存目录

1. .hvigor/cache
编译全局缓存文件夹，包含file-cache.json、meta.json、task-cache.json，记录编译文件、任务缓存，大幅提升本地二次编译速度。
2. .hvigor/dependencyMap
本地依赖解析映射目录

- dependencyMap.json5、oh-package.json5：本机环境专属的依赖路径、依赖关系缓存，换电脑、换环境就会失效。

3. .hvigor/outputs
项目编译全部输出目录

- build-logs/build.log：完整编译过程日志，本地排查编译报错专用
- logs/details：运行、编译详细日志记录
- sync/fileCache.json、output.json：IDE工程同步临时中间文件

4. .hvigor/report
编译完成后自动生成的代码检查、性能分析、质量报告JSON文件，仅本机查看调试使用。

四、.idea IDE专属配置目录

1. .idea/.deveco
DevEco Studio编辑器专属本机配置

- cxx、module：C++环境、模块本地配置

2. .idea/modules
IDE本地模块索引缓存

- DecodePlayControl.iml、entry.iml：IDE模块识别本地配置文件
- project.cache.json：IDE本地工程缓存

五、entry内 .cxx 原生编译缓存

1. .cxx/default/default/debug/arm64-v8a
C/C++原生代码架构编译目录，适配手机arm架构处理器
2. .cmake / CMakeFiles
Cmake编译构建规则、中间缓存目录
3. .ninja_deps / .ninja_log / build.ninja
Ninja编译构建工具的依赖、日志、构建脚本本地文件
4. additional_project_files.txt / build_file_index.txt
编译过程临时记录文件
5. cmake_install.cmake / CMakeCache.txt
Cmake环境缓存、安装配置文件
6. compile_commands.json / hvigor_native_config.json
原生代码编译命令、原生构建本地配置缓存
7. native_work_dir.txt / output.log
原生编译工作目录日志文件

六、build 编译产物目录

1. build/config
编译全局基础配置缓存buildConfig.json
2. build/default/cache/generated/profile
编译过程生成的缓存、资源、配置文件
3. build/default/intermediates
编译中间产物目录

- cmake、hap_metadata、libs、loader、merge_profile、package、patch等：打包过程中间处理文件

4. build/default/outputs/default
最终打包输出目录

- mapping：代码混淆映射文件
- entry-default-unsigned.hap：项目最终生成的可安装APP安装包
- pack.info：安装包打包信息文件

七、oh_modules 

1. oh_modules/libplayer.so
引入的第三方播放器原生库文件
2. Index.d.ts
第三方库TS类型声明文件
3. oh-package.json5
第三方子模块依赖配置
4. oh_modules/.ohpm/lock.json5
OHPM包管理器本地全局锁定缓存

八、全局通用忽略垃圾文件

- .cxx 全部编译缓存
- .clangd / .clang-format / .clang-tidy 代码格式化、语法检查本地配置
- .appanalyzer 应用分析工具缓存
- *.log 全部日志文件
- local.properties 本机SDK路径隐私本地配置
- **/.test 本地单元测试临时文件
- .DS_Store / Thumbs.db 系统自动生成无用隐藏文件
